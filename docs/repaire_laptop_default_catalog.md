# Repaire Laptop Default Catalog

Purpose: default laptop catalog for the repair workflow.

UX rule:
- Show common families first.
- Put old/rare families under “More models”.
- Always provide search.
- Always provide “Other / Add custom model”.
- Do not force exact model selection.
- Flow should be: Laptop → Brand → Family → Common variant/series → exact model typed manually if needed.

Coverage target:
- Around 70% of common repair cases by default.
- Remaining cases handled by custom brand/model/manual entry.

# Laptop Repair Flow

Recommended repair selection:

1. Select device family: Laptop
2. Select brand
3. Select family
4. Select common variant/series
5. Enter exact model manually if needed
6. Select problem
7. Suggested parts appear
8. Add inventory part or manual part
9. Create ticket

Fallbacks:

- Other brand
- Other model
- Add custom brand
- Add custom family
- Add custom model
- Add manual problem
- Add manual part

# Data Model Notes

Recommended tables:

- device_families
- device_brands
- device_model_families
- device_model_variants
- device_models_custom
- repair_problem_categories
- repair_problems
- part_categories
- parts
- part_compatibility
- store_custom_catalog_entries

Important:
- Default catalog is global seed data.
- Store custom entries are tenant/store-specific.
- Keep defaults editable only by super admin.
- Store additions should not modify global catalog.
